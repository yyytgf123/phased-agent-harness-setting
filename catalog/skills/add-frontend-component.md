# 신규 프론트엔드 컴포넌트 추가

## 언제 쓰나 / 안 쓰나
- 트리거: 새 컴포넌트/페이지/위젯 신설
- 비트리거(near-miss): 기존 컴포넌트 리스타일/문구 변경 → 직접 수정

## 범위 제약
- 수정 가능: `app/` 프론트엔드 소스만
- 절대 금지: 백엔드/인프라 범위 수정

## 절차

### Step 1 — 실패 컴포넌트 테스트 먼저 (TDD)
유사 컴포넌트 패턴 확인 후, 렌더·상호작용·상태 전이를 검증하는 컴포넌트 테스트를 **먼저 작성**(RED).
- 검증: `npm test -- NewComponent` (RED 확인)

### Step 2 — 구현
컴포넌트 구현. 로딩/에러/빈 상태와 키보드 접근성(aria/role/focus) 포함.
- 검증: `npm run build`

### Step 3 — GREEN + 린트
테스트 통과까지 구현, a11y 단언 포함.
- 검증: `npm test -- NewComponent`

### Step 4 — 완료 검증 (필수)
- [ ] `npm test` 통과
- [ ] 범위 밖 파일 수정 없음
- [ ] 접근성 및 로딩/에러/빈 상태 처리 확인

## 위험 작업 처리 (해당 시)
- apply/migrate/secret은 plan/dry-run까지만. 전체 금지목록·표준 응답은 루트 헌법 `claude.md`의 `# CRITICAL — Safety` 참조(복붙 금지).

# 버그 수정 (재현 테스트 우선)

## 언제 쓰나 / 안 쓰나
- 트리거: 잘못된 동작/예외/엣지케이스 결함 수정(예: 결제 멱등성 깨짐, 주문 상태 전이 오류)
- 비트리거(near-miss): 오타·주석·로그 문구 변경 → 직접 수정

## 범위 제약
- 수정 가능: `app/` 만
- 절대 금지: 다른 범위 수정, 테스트 없는 "그냥 픽스"

## 절차

### Step 1 — 실패하는 재현 테스트 먼저 (TDD)
버그를 정확히 드러내는 JUnit5 테스트를 **먼저 작성**한다. 반드시 RED가 떠야 재현이 입증된 것이다.
- 검증: `./gradlew test --tests '*BugReproTest'` (RED 확인)

### Step 2 — 최소 수정
근본 원인만 고친다. 무관한 리팩터 금지.
- 검증: `./gradlew compileJava`

### Step 3 — GREEN + 회귀 보존
재현 테스트가 통과할 때까지 수정, 회귀 테스트로 남긴다.
- 검증: `./gradlew test --tests '*BugReproTest'`

### Step 4 — 완료 검증 (필수)
- [ ] `./gradlew test` 전체 통과(다른 테스트 깨짐 없음)
- [ ] 범위 밖 파일 수정 없음
- [ ] 재현 테스트가 픽스 없이는 실패함을 확인

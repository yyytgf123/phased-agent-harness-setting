# Phase 0 — 스택 확정 (Setup)

> 본 워크플로 시작 전, 사용자 스택과 버전을 먼저 못 박는다.
> 이게 안 되면 뒤 단계가 추측으로 흐른다.
> 사용자 입력은 별도 파일이 아니라 **루트 `ORCHESTRATOR.md`의 입력 칸**으로 받는다.

## 할 일
1. `ORCHESTRATOR.md`에 사용자가 적은 "사용 도구 + 버전 + 도메인 설명"을 읽는다.
2. 버전이 비어 있으면 `version-policy.md`로 호환 버전을 **검색 후 선정**한다.
3. 선정 버전을 **표로 출력**하고 사용자 확인을 받는다.
4. 확인되면 → `../phase1_discovery/`로 진행.

## 이 폴더 파일
| 파일 | 용도 |
|------|------|
| `version-policy.md` | 버전 미기재 시 호환 버전 검색·선정 규칙 |
| `tooling-matrix.md` | 도구 → 검증 명령 매핑 (뒤 Phase에서도 계속 참조) |

> 사용자가 채우는 파일은 이 폴더에 없다. 입력은 루트 `ORCHESTRATOR.md`에서.

## 쓸 템플릿 (`../../templates/`)
| 템플릿 | 구축 결과 |
|--------|-----------|
| `reports/version-table.md.tmpl` | 확정 버전 표 (사용자 승인용) |

## 입력 / 출력
- 입력: ORCHESTRATOR.md의 사용 도구·버전·도메인 설명
- 출력: 확정된 버전 표 + repo_type(app/infra/mixed) → Phase 1로 전달

## 게이트
버전 표를 사용자가 승인하기 전에는 Phase 1로 넘어가지 않는다.

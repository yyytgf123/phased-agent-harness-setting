# Phase 1 — 도메인 분석 (Discovery)

> 레포를 탐색만 한다. **파일 생성 금지, 보고만.**
> AI가 추측으로 하네스를 만들지 못하게 막는 관문.

## 할 일
1. 레포가 **앱 전용 / 인프라 전용 / 혼합** 중 무엇인지 판정.
2. 핵심 작업 유형 식별 (앱: 기능·버그·리팩터·테스트 / 인프라: 모듈·배포·알림·대시보드).
3. 기존 `.claude/` 확인 — 충돌·중복 방지.
4. `discovery-checklist.md`의 항목대로 조사 → 표로 보고.
5. 사용자 숙련도 감지 → 톤 조절.

## 이 폴더 파일
| 파일 | 용도 |
|------|------|
| `discovery-checklist.md` | 조사할 6개 항목 + 보고 결과를 뒤 Phase에서 어떻게 쓰는지 |

## 쓸 템플릿 (`../../templates/`)
| 템플릿 | 구축 결과 |
|--------|-----------|
| `reports/discovery.md.tmpl` | 탐색 보고서 (파일 생성 아님 — 보고 골격) |

## 입력 / 출력
- 입력: Phase 0의 확정 스택
- 출력: 탐색 보고서(명령·디렉터리지도·컨벤션·위험지대·기존설정·분류) → Phase 2로

## 원칙
- 추측한 부분은 "추측"으로 표시.
- 어떤 파일도 만들거나 고치지 않는다.

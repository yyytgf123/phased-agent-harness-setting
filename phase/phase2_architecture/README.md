# Phase 2 — 팀 아키텍처 설계 (Architecture)

> 누가(에이전트) 어떤 구조로 협업할지 정한다. 아직 파일은 안 만든다(설계만).

## 할 일
1. **실행 모드 선택**: 에이전트 팀(기본) vs 서브 에이전트.
   - 2명+ 협업 → 팀. 1명이거나 통신 불필요 → 서브 에이전트.
2. **아키텍처 패턴 선택** (6패턴 중): 개발/인프라는 보통 **생성-검증(Producer-Reviewer)** 기본.
3. **에이전트 분리** (4축: 전문성·병렬성·컨텍스트·재사용성).
   - 혼합 레포는 **컨텍스트 축이 결정적** → backend-dev / infra-dev 반드시 분리.

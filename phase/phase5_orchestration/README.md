# Phase 5 — 통합 및 오케스트레이션 (Orchestration)

## 이 단계는 무엇을 하나
개별 에이전트·스킬을 하나의 워크플로로 엮는 오케스트레이터 스킬을 구축한다.
"누가 언제 어떤 순서로 협업하는가"를 정의한다.
- 오케스트레이터 → `프로젝트/.claude/skills/{orchestrator}/SKILL.md`

## 읽을 상세 (이 폴더)
| 파일 | 무엇을 담음 |
|------|-------------|
| `orchestrator-template.md` | 모드별 패턴, 데이터 전달표, 에러 전략, 팀 크기 |

## 쓸 템플릿 (`../../templates/`)
| 템플릿 | 구축 결과 |
|--------|-----------|
| `SKILL.md.tmpl` | 오케스트레이터도 스킬의 한 형태 — 같은 골격 사용 |

## 구축 절차
1. `orchestrator-template.md`를 읽어 실행 모드에 맞는 패턴을 고른다.
   - 팀: TeamCreate → TaskCreate → 자체조율(SendMessage) → 종합 → 정리.
   - 서브: Agent 직접 호출(run_in_background) → 수집 → 통합.
2. 데이터 전달 프로토콜 명시 (태스크 + 파일 `_workspace/` + 메시지 조합).
3. 에러 핸들링 (1회 재시도 후 누락 명시, plan 차이/apply 차단/범위침범 처리).
4. 팀 크기 점검 (소 2~3 / 중 3~5 / 대 5~7).
5. 테스트 시나리오 작성 (정상 1 + 에러 1 이상).

## 입력 / 출력
- 입력: Phase 3 에이전트 + Phase 4 스킬
- 출력: 오케스트레이터 스킬 1개 (데이터 흐름 + 에러 + 테스트 시나리오 포함) → Phase 6로

## 게이트
- 데이터 경로에 dead-link 없는가 (쓰는 파일을 읽는 쌍이 맞는가).
- Phase 순서가 논리적인가 (구현 전에 배포 plan 나오지 않는가).
